De WordPress A Jekyll

WordPress est en 2015 la plateforme de référence pour la publication web. Du simple blog à des sites d'entreprise, WordPress s'est imposé comme LA solution de référence et le CMS numéro un du moment.

Mais pour toutes ses qualités, WordPress n'est qu'une solution web parmi d'autres. Et la réflexion continue sans cesse.

Face à l'hégémonie des CMS conçus en php, une nouvelle approche prend de l'ampleur pour la publication de sites personnels sur le modèle du blog: il s'agit des générateurs de sites statiques. Et la référence de la catégorie s'appelle Jekyll.

L'idée derrière Jekyll (et des générateurs de sites statiques en général) est de tenter une approche hybride entre l'ancienne formule des sites purement satiques et les CMS bien plus complexes. D'un côté on conserve un thème commun à toutes les pages, un ou plusieurs modèles de pages et surtout la séparation claire entre le contenu, la structure et l'apparence du site. A l'autre extremité on retrouve des pages purement statiques composées de html de css et de javascript sans base de données et sans code exécuté sur le serveur (pas de php ou équivalent).

Pourquoi remettre en question les cms ?

Le principal inconvénient des CMS pour sites personnels est leur relative complexité. Un site wordpress exécute du code php, fait de nombreuses requêtes sollicitant le serveur de bases de données et se compose d'un nombre relativement important de fichiers. Le code côté serveur est non seulement pénalisant en terme de performance mais il est surtout la principale source de problèmes de sécurité.

Fonctionnement de Jekyll

Jekyll est une solution écrite en Ruby qui utilise le language Markdown pour la création du contenu (apr exemple des articles) et Liquid pour les templates. Aucune connaissance en ruby n'est réellement nécessairement pour utiliser Jekyll, tout comme peu d'utilisateurs de wordpress ont besoin de connaître php. Ruby se révèle surtout util pour la cration de plugins Jekyll... et pur ceux qui souhaitent contribuer au projet. Liquid utilise des balises spécifiques et des filtres pour afficher du contenu comme par exemple:

 titre_page  site.name

Pour la création des articles (posts), il suffit de créer un fichier par article. Idéalement le nom du fichier contient la date de ce dernier afin d'aider au classement chronologique. L'en-tête de l'article est constitué de meta-informations comme le type de contenu (layout:) ou l'image utilisée pour la liste d'articles.

Par exemple l'article que vous lisez en ce moment contient l'en-tête suivant:

comments: true
date: 2013-11-01 12:00:00
layout: post
slug: wordpress-a-jekyll
title: De WordPress A Jekyll
summary: 'Jekyll est peut-être la bonne solution pour la gestion de mon blog...'
image: 'wordpress-a-jekyll/jekyll.jpg'
tags:
- presence web
- jekyll
- cms

Pour le reste de l'article, la rédaction se fait à l'aide de Markdown ce qui permet d'utiliser un simple editeur de texte tout en bénéficiant d'options de mise en forme de texte.
